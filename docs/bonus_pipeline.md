# Bonus Spec — Feedback-Driven AI Training Pipeline

Our application collects explicit user feedback (UP/DOWN votes) along with user preferences and the specific content snippet generated by the AI. This data drives a continuous optimization loop for our AI Crypto Advisor.

## 1. Short-Term: Few-Shot In-Context Learning (Immediate Calibrations)
Instead of running expensive retraining pipelines daily, we use the stored feedback for real-time prompt engineering:
- **Data Aggregation:** A background worker runs every 6 hours, aggregating the most highly upvoted AI Insights, segmented by `investorType` (e.g., HODLer, Day Trader).
- **Dynamic Prompt Injection:** When a user requests a daily insight, our `ai.service.ts` fetches the top 2-3 historically upvoted insights for that specific investor type and injects them into the LLM system prompt as examples:
  ```text
  "Here are examples of market summaries users with this investment profile previously upvoted:
   - [Upvoted Example 1]
   - [Upvoted Example 2]
   Please generate today's insight using a similar tone and analytical depth."
